# Academic CV Template

This repository contains a Quarto-based academic CV template designed for a polished promotion, tenure, or faculty dossier. It renders to PDF and DOCX, keeps content in small section files, and uses restrained black-and-white styling optimized for print.

The current example is my CV, but the structure is reusable for another faculty CV.

## What This Template Provides

- A single Quarto source document that assembles modular CV sections.
- PDF output styled like a traditional R1 faculty CV, not a website.
- DOCX output using a Word reference document.
- Generated career-highlight metrics for quick dossier scanning.
- Separate files for styling, filters, and content.
- A GitHub Actions workflow that renders PDF and DOCX deliverables.

## Requirements

Install these before rendering locally:

- [Quarto](https://quarto.org/docs/get-started/)
- Python 3
- A LaTeX distribution for PDF output, such as TinyTeX
- `make`, optional but recommended

If Quarto is installed, TinyTeX can usually be installed with:

```sh
quarto install tinytex
```

## How To Reuse This For Your CV

1. Fork or copy this repository.
2. Rename `nelson-roque-tenure-cv.qmd` if desired.
3. Update the `title` in the YAML header of the main `.qmd` file.
4. Replace the content in `sections/*.qmd` with your own CV content.
5. Keep section files small and topic-specific.
6. Update `scripts/generate_career_highlights.py` if your section names or metrics differ.
7. Update `Makefile` and `.github/workflows/render-cv.yml` if you rename the main `.qmd` file or output filenames.

Avoid editing `sections/career-highlights.qmd` directly. It is regenerated by `scripts/generate_career_highlights.py`.

## Editing Content

Most CV content lives in `sections/`.

Use ordinary Quarto Markdown where possible:

- Use `##` for major sections.
- Use `###` or `####` for subsections.
- Use numbered lists for publications and presentations.
- Use bold text only for emphasis that helps reviewers scan.
- Keep URLs and DOI links in the source even if the PDF style de-emphasizes them.

The main file controls ordering:

```qmd
{{< include sections/appointments.qmd >}}
{{< include sections/career-highlights.qmd >}}
{{< include sections/education.qmd >}}
```

Move sections by reordering the include lines in the main `.qmd`.

## Career Highlights

Career highlights are computed by:

```sh
python3 scripts/generate_career_highlights.py
```

The script currently counts or summarizes:

- Peer-reviewed publications
- Books / chapters
- Current external funding
- Total external funding
- Invited talks
- NIH grants
- Program leadership roles
- NIH reviewer service
- Major media coverage

If adapting this template, review the script carefully. It expects specific section files and formatting patterns, especially numbered publication lists and grant blocks marked with `::: {.grant}`.

## Rendering Locally

Render both PDF and DOCX:

```sh
make render
```

Or render manually:

```sh
python3 scripts/generate_career_highlights.py
quarto render nelson-roque-tenure-cv.qmd --to pdf --output-dir rendered
quarto render nelson-roque-tenure-cv.qmd --to docx --output-dir rendered
```

Outputs are written to `rendered/`.

## Styling

PDF styling is controlled mainly by:

```text
styles/cv.tex
```

HTML and general visual styling are controlled by:

```text
styles/cv.css
```

The design is intentionally conservative:

- Black and white only
- No cards, icons, gradients, shadows, or decorative elements
- Slightly increased line spacing
- Thin rules beneath major headings
- Hanging-indent style lists for scholarly entries
- Print-focused spacing and page-break behavior

## GitHub Actions

The workflow in `.github/workflows/render-cv.yml` renders the CV on pushes to `main` and on manual workflow dispatch. It uploads the PDF and DOCX as workflow artifacts.

If you rename files, update the workflow paths so GitHub Actions renders the correct `.qmd` file and uploads the correct files from `rendered/`.

## Reuse Checklist

Before sharing your adapted repository:

- Update the main `.qmd` title.
- Replace all personal content in `sections/`.
- Check `scripts/generate_career_highlights.py` against your section structure.
- Rename output files in the `Makefile` and GitHub Actions workflow.
- Run `make render`.
- Open the generated PDF and inspect page breaks, spacing, and headings.
- Confirm `rendered/` contains the current PDF and DOCX.
